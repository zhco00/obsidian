SEO(Search Engine Optimization) -> 검색엔진의 검색결과에 자신의 웹 페이지가 더 많이 노출되도록 최적화된 컨텐츠를 만드는 것
SEO체크 사이트
https://www.seobility.net/en/seocheck/

즉
웹 크롤러(크롤링)이라는것은 검색엔진이 웹 페이지를 찾고 인덱싱하는 과정중 하나이다.
구글봇 또는 스파이더하고 불리는 특수 프로그램을 사용하여 웹을 순회하고 콘텐츠를 수집한다.
즉 seo를 한다는것은 웹 크롤링의 찾는 과정에서 잘 찾을 수 있게 최적 하는 것이라고 볼 수 있나?

구글 검색엔진의 단계

1. 크롤링
	1. 웹에 어떤 페이지가 존재하는지 파악하는 것입니다
	   크롤러라는 자동화된 프로그램으로 계속해서 신규 또는 업데이트된 페이지를 찾습니다. 그런 다음 나중에 볼 수 있도록 발견된 페이지의 주소 (또는 페이지 URL)을 큰 목록에 저장합니다. 
2. 색인(인덱싱)
	1. 크롤링을 통해 파악한 페이지를 방문하여 각 페이지에 대한 정보를 분석합니다. 페이지의 컨텐츠, 이미지, 동영상 파일을 분석하여 어떤 페이지인지 파악하려고 합니다. 이 정보는 수많은 컴퓨터에 저장된 거대한 데이터베이스인 구글색인에 저장됩니다.
3. 검색 결과 반영
	1. 사용자가 검색하면 구글에서는 가장 높은 품질의 검색결과를 결정하려고 합니다. ‘최고의’ 검색결과에는 사용자 위치, 언어, 기기(데스크톱 또는 휴대전화), 이전 검색어를 비롯한 여러 요소가 있습니다.

구글 서치 콘솔를 제공하여 웹사이트 소유자들이 인덱싱 요청을 할 수 있도록 합니다. 

이미지 Alt 태그는 구글 크롤러에게 해당 이미지가 어떤 내용의 이미지 인지 텍스트로써 설명을 하는 것입니다.

인덱싱 요청 -> https://search.google.com/search-console/about
구글 서치 콘솔(Google Search Console)은 구글에 웹사이트를 검색했을 때 어떻게 보여지는지 그리고 어떻게 점검해야 하는지 도와주는 무료 플랫폼입니다.

구글 서치 콘솔을 [사용해야 하는 이유](https://support.google.com/webmasters/answer/9128668?hl=ko)를 구글에서는 아래처럼 말합니다:

- 구글의 웹사이트 노출 횟수 및 크롤링 가능 여부 확인

- 색인 상태 점검 및 문제 개선

- 트래픽을 유발하는 관련 페이지 및 검색 키워드 분석

- 사이트의 스팸 외 기타 문제 발견 여부

- 관련된 사이트 분석

- AMP (Accelerated Mobile Page), 모바일 사용 편의성 및 기타 검색 관련 문제 해결
